# Contributing to END2DALY

We welcome good contributions! This is a community effort and we appreciate your interest in helping improve this project.

## ⚠️ Important Notice

**This is an active work-in-progress project.** Please be aware that:
- Functions, APIs, and implementation details may change without notice
- The project structure is still evolving
- Breaking changes may occur between updates

We disclaim responsibility for any issues arising from using this project.

## How to Contribute

We encourage contributions of all kinds! Please follow this workflow:

### 1. Start with an Issue
Before making changes, please **open an issue** to discuss your proposed contribution:
- Describe what you want to contribute
- Explain the problem it solves or the feature it adds
- Get feedback from maintainers and the community

### 2. Fork and Create a Feature Branch
Once the issue is discussed and approved:
1. Fork the repository
2. Create a feature branch based on the issue: `git checkout -b feature/YourFeature`
   - Reference the issue number in your branch name if possible

### 3. Make Your Changes
- Make your changes in your feature branch
- Keep commits clear and descriptive
- Link your commits to the related issue

### 4. Commit and Push
1. Commit your changes: `git commit -am 'Add some feature (fixes #123)'`
2. Push to your fork: `git push origin feature/YourFeature`

### 5. Create a Pull Request
1. Open a Pull Request against the main repository
2. Reference the issue(s) your PR addresses
3. Describe your changes clearly
4. Wait for review and feedback

### 6. After Approval
After your Pull Request has been accepted and merged, you can clean up your local fork.

## Guidelines

- Keep contributions focused and manageable
- Write clear commit messages
- Be patient with review feedback
- Test your changes if applicable
- Update documentation if needed

Thank you for contributing to END2DALY!
