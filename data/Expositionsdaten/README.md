# Expositionsdaten (Noise Exposure Data)

## Folder Structure

- **`raw/MetadatenExpositionsdaten.csv`**: Master registry of all raw Excel files
  - **To be manually maintained**; update this when adding new data sources
  - Description in **`raw/Beschreibung MetadatenExpositionsdaten.txt`**
  
- **Raw data files** (to be listed in metadata):
  - `ColNamesStr_*.txt` (Column name text files) **to be updated manually**
  - `BW_Belastungsstatistik_2022.xlsx` (Baden-Württemberg)
  - `EU_END_2022_cutoff241118_v4.xlsx` (EEA; provided by EIONET)
  - `HE_ULK2022StatistikGesamt_20240704.xlsx` (HLNUG)
  - `NRW_Ballungsräume_Runde_4_2025.xlsx` 
  - `NRW_nicht_Ballungsräume_2023_07_20.xlsx` 
  - `NS_2024-10-25_END422_Statistiken_Gemeinden_Ergebnisblatt_V4.xlsx` 

  
- **`processed/`** (content might be gitignored): Cleaned & processed data
  - `allExpo.csv`: Unified long-format exposure data for all sources
  - Generated by `01_prepare_exposure_data.qmd`
  
## Processing

Run: `Expositionsdaten_vorbereiten.qmd`

**What it does:**
1. Reads metadata registry
2. For each dataset: reads Excel, applies column names, filters
3. Transforms to long format (one row per municipality × metric × exposure band x metric x source x mapping extend)
4. Standardizes existing 8-digit geocodes
5. Enriches with geocodes from Destatis data if they are missing
6. Exports to `processed/allExpo.csv`

## Notes

- All processed data in `processed/` is reproducible from scripts. Never edit by hand
- Column name files (`.txt`) are manual; update if Excel format changes
